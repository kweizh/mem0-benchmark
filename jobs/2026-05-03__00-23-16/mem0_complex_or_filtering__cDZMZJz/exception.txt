# Mem0 Complex OR Filtering

## Background
Mem0 partitions memories by `user_id`, `agent_id`, `app_id`, and `run_id`. When building multi-agent systems, you often need to retrieve memories that belong to EITHER the user OR the agent. Using an AND logic for these IDs will return nothing because records are stored per-entity. You need to use an OR filter.

## Requirements

1. Initializes the Mem0 OSS `Memory` client. (It will automatically use the default Qdrant in-memory store and OpenAI for embeddings/LLM using the `OPENAI_API_KEY` from the environment).
2. Adds a memory: "Alice has a knee injury" for `user_id="alice"`.
3. Adds a memory: "The coach recommends swimming for recovery" for `agent_id="coach"`.
4. Searches for the query "recovery" across BOTH `user_id="alice"` AND `agent_id="coach"` using a compound `$or` filter in the `filters` argument. Do not pass `user_id` or `agent_id` as direct keyword arguments to the `search` method.
5. Saves the search results (a list of dictionaries) to `/home/user/results.json` using the standard `json` module.
